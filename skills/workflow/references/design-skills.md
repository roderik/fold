# Design Skills Reference

Run `/impeccable:teach-impeccable` once per project to establish design context.

| Skill | Purpose |
|-------|---------|
| `/impeccable:polish` | Final quality pass (alignment, spacing, consistency) |
| `/impeccable:audit` | Technical quality (a11y, perf, theming, responsive) |
| `/impeccable:critique` | UX evaluation with scoring |
| `/impeccable:typeset` | Typography (fonts, hierarchy, sizing, weight) |
| `/impeccable:arrange` | Layout, spacing, visual rhythm |
| `/impeccable:overdrive` | Ambitious effects (shaders, physics, 60fps) |
| `/impeccable:bolder` | Amplify bland designs |
| `/impeccable:quieter` | Tone down aggressive designs |
| `/impeccable:colorize` | Add strategic color |
| `/impeccable:distill` | Strip to essence |
| `/impeccable:harden` | Error handling, i18n, edge cases |
| `/impeccable:normalize` | Realign to design system standards |
| `/impeccable:animate` | Purposeful animations and micro-interactions |
| `/impeccable:adapt` | Responsive design, breakpoints, cross-device |
| `/impeccable:delight` | Moments of joy and personality |
| `/impeccable:onboard` | Onboarding flows and first-run experiences |
| `/impeccable:extract` | Extract reusable components and design tokens |
| `/impeccable:clarify` | Improve UX copy, error messages, labels |
| `/impeccable:frontend-design` | Create distinctive, production-grade UI |
